### Cocos測驗

#### 　如何優化 Cocos Creator 包體體積

##### 如何優化圖片文件體積?

1. 圖片壓縮：對於圖片來說，最直接的方法就是將圖片進行壓縮，以減少圖片的體積。壓縮的方式有很多種，例如使用JPEG、PNG、WebP等格式進行壓縮。

   {% note info%}

   在壓縮的過程中，需要平衡圖片的品質和體積大小，避免過度壓縮影響圖片的清晰度。

   {% endnote %}

2. 圖片格式轉換：不同的圖片格式有不同的壓縮方式，對於需要優化的圖片，可以試著將其轉換為體積更小的格式。例如將一些使用PNG格式的圖片轉換為使用JPEG格式，或者將一些使用GIF格式的動圖轉換為使用APNG或WebP格式。

3. 圖片尺寸縮小：對於需要使用的圖片，可以根據實際需求調整圖片的尺寸，減少圖片的 pixel 數量，進而減小圖片的體積。需要注意的是，在調整圖片尺寸時，需要保持圖片的比例不變，避免圖片變形。

4. 圖片精簡：對於一些不需要使用的像素，可以使用圖片編輯軟體將其刪除，減少圖片的大小。例如將一些無用的背景像素刪除，或者將一些重複的元素進行合併。

5. 使用向量圖（SVG）：矢量圖是一種使用數學公式描述圖形的圖片格式，它的優點是可以無限放大而不失真，同時也可以減少圖片的體積。如果可以使用矢量圖替代某些圖片，就可以進一步減小文件的體積。

   -------

##### 如何優化聲音文件體積

1. 聲音壓縮：對於聲音文件，最直接的方法就是將其進行壓縮，以減少聲音的體積。壓縮的方式有很多種，例如使用MP3、OGG、AAC等格式進行壓縮。需要注意的是，在壓縮的過程中，需要平衡聲音的品質和體積大小，避免過度壓縮影響聲音的音質。

2. 降低比特率 （Bitrate）：對於聲音文件，可以進行比特率的調整，降低聲音文件的品質以減小其體積。需要注意的是，降低比特率會對聲音品質產生影響，需要在保持聲音品質可接受的情況下進行調整。

3. 精簡聲音內容：對於一些不需要使用的聲音內容，可以使用聲音編輯軟體將其刪除，減少聲音的大小。例如將一些無用的背景聲音刪除，或者將一些重複的元素進行合併。

4. 使用短片段：對於一些長時間的聲音文件，可以根據實際需求選擇其中較小的片段使用，進而減小聲音的體積。

5. 節省循環次數：對於一些需要循環播放的聲音，可以減少其循環播放的次數，減少聲音的體積。

##### seeting.js大小與什麼相關

`settings.js`是 Cocos Creator 項目的設定檔案，用來定義遊戲的基本設定，例如遊戲場景的尺寸、顯示模式、音量等等。`settings.js`的大小會與遊戲設定有關，包括場景大小、繪製設定、預載資源等等。

以下是一些可能會影響`settings.js`大小的因素：

1. 場景大小：如果場景較大，`settings.js` 中記錄的相關配置也會相應增加，從而導致文件大小增大。

2. 預載資源：如果項目中需要預載大量資源，例如圖片、音頻等，相關配置也會出現在 `settings.js` 中，從而導致文件大小增大。

3. 引擎版本：不同版本的 Cocos Creator 引擎中，settings.js 文件的大小可能會有所不同。例如在某些版本中可能會增加一些新的配置項，從而增大文件大小。

4. 程式碼注入：一些第三方插件或者自定義程式碼可能會在運行時動態地修改 `settings.js` 的內容，從而導致文件大小增大。

總的來說，`settings.js` 的大小主要取決於遊戲的設定和所使用的引擎版本等因素，需要在開發過程中注意優化設定，避免不必要的設置和冗余資訊，從而減小 `settings.js` 的大小。

##### 如何優化引擎程式碼體積

1. 移除未使用的模組和功能：如果你使用的引擎中包含一些你沒有使用的模組或功能，可以考慮將其移除，從而減小程式碼體積。
2. 使用模組化的開發方式：使用模組化的開發方式可以幫助你更好地組織程式碼，避免不必要的重複程式碼，從而減小程式碼體積。
3. 使用程式碼壓縮工具：可以使用程式碼壓縮工具，例如 UglifyJS，來壓縮程式碼體積。
4. 避免冗余程式碼：避免寫冗余的程式碼，例如重複的函數定義或類定義等，可以減小程式碼體積。
5. 優化程式碼邏輯：優化程式碼邏輯，例如減少循環次數、避免過多的判斷、使用更高效的算法等，也可以幫助減小程式碼體積。
6. 使用 Tree Shaking 技術：Tree Shaking 技術可以自動移除未使用的程式碼，從而減小程式碼體積。

##### 如何優化字體字庫文件大小

1. 壓縮字體文件：可以使用壓縮工具，例如 Zopfli 或 Brotli，將字體文件進行壓縮，從而減小文件大小。
2. 子集化字體：只保留使用到的字型，將字體文件子集化，從而減小字體文件大小。可以使用 Font Squirrel 的子集化工具或 Fontmin 進行字體子集化。
3. 優化字體編碼：可以使用高效的編碼方式，例如 WOFF2，來優化字體編碼，從而減小字體文件大小。
4. 選擇合適的字體格式：根據具體的使用情境，選擇合適的字體格式，例如 TTF、OTF、WOFF、WOFF2 等，可以減小字體文件大小。
5. 減小字體文件中的元數據：字體文件中通常包含一些元數據，例如作者信息、版權信息、設計風格等，可以通過移除這些元數據，從而減小字體文件大小。

#### 如何做資源管理

##### cc.Script 如何顯示遠端伺服器上的圖片

要在 Cocos Creator 中顯示遠端伺服器上的圖片，可以使用 cc.Sprite 组件。

1. 在 Cocos Creator 中創建一個新的場景或者打開現有的場景。
2. 在場景中創建一個 cc.Sprite 組件。選中該組件，進入屬性檢查器。
3. 在屬性檢查器中找到 SpriteFrame 屬性，點擊右側的齒輪圖標，進入 Sprite Editor 編輯器。
4. 在 Sprite Editor 編輯器中，點擊左側的「圖片素材」選項卡，選擇「從 URL」。
5. 在 URL 欄位中輸入遠端伺服器上的圖片地址，並按下 Enter 鍵。
6. Cocos Creator 會自動載入圖片，並顯示在 Sprite Editor 編輯器的預覽區域中。可以調整圖片的裁剪、尺寸和位置等屬性。
7. 編輯完成後，點擊右下角的「保存」按鈕，將圖片保存為 SpriteFrame。
8. 回到屬性檢查器中的 SpriteFrame 屬性，選擇剛剛保存的 SpriteFrame。
9. 最後，選擇預覽該 cc.Sprite 組件，即可在遊戲中顯示遠端伺服器上的圖片。

{% note info%}

在載入遠端伺服器上的圖片時，需要確保圖片的網路可用性和適當的授權權限。另外，載入大型圖片可能會影響遊戲性能，可以考慮使用圖片縮放和壓縮等技術來進行優化。

{% endnote %}

##### 包體體積超過 4M 如何處理

1. 圖片、音效、字體等資源壓縮：對於大型的圖片、音效、字體等資源，可以使用相應的壓縮工具，如 TinyPNG、Audacity 等，將其壓縮至最小，以減少包體體積。
2. 刪除無用資源：檢查遊戲中是否存在未使用的資源，如圖片、音效等，將其從遊戲包中刪除，以減少包體體積。
3. 使用動態下載：將一些大型資源，如場景、圖片、音效等，分離出來，使用動態下載的方式進行載入。這樣可以將遊戲包體積縮小至 4M 以下，同時也可以降低首次載入時間。
4. 使用分包機制：將一些較為獨立的模塊或資源打包成獨立的小包，使用分包機制進行動態下載。這樣可以將主包體積控制在 4M 以內，同時也可以提高遊戲載入速度和運行效率。
5. 使用 CDN：將遊戲資源上傳到 CDN，使用 CDN 進行資源載入。這樣可以減少伺服器的壓力，同時也可以提高遊戲載入速度和運行效率。

1. 使用雲服務提供商：現在有很多雲服務提供商（如 AWS、Azure、阿里雲等）提供了雲服務部署解決方案，可以使用這些雲服務提供商提供的 API 和工具進行部署。

2. 自建遠端伺服器：如果你有自己的伺服器，可以在上面部署你的遠端資源。你需要使用一些相關的軟件，如 FTP、SFTP、SCP 等，將你的資源文件上傳到伺服器上，並通過 API 或者網頁訪問進行操作。

3. 使用第三方平台：現在有一些第三方平台，如 GitHub、GitLab、Bitbucket 等，可以作為你的遠端資源部署的儲存庫。你可以在這些平台上建立一個專案，將你的資源文件上傳到這個專案中，然後通過相應的 API 進行操作。

在進行第三方遠端資源部署的過程中，需要對相應的資源進行權限控制和安全加密，以保障資源的安全性。同時，也需要適當地選擇遠端資源的存儲位置和方式，以提高資源的存取速度和運行效率。

##### loadRes 與 load 有何區別

在 Cocos Creator 中，`cc.loader.loadRes` 和 `cc.loader.load` 都是用來載入資源的方法，但是它們之間有一些區別：

1. 載入路徑不同：`cc.loader.loadRes` 載入的是 resources 資料夾下的資源，而 `cc.loader.load` 可以載入任何路徑下的資源，例如 <http://> 或 file:// 等。
2. 參數傳遞方式不同：`cc.loader.loadRes` 只需要傳遞資源的相對路徑即可，而 `cc.loader.load` 則需要傳遞一個包含資源路徑和資源類型等參數的對象。
3. 緩存機制不同：`cc.loader.loadRes` 載入的資源會被緩存到記憶體中，可以多次使用，而 `cc.loader.load` 載入的資源預設是不會被緩存的，每次都需要重新載入。

如果需要載入 resources 資料夾下的資源，建議使用 `cc.loader.loadRes` 方法，因為它比較簡單、方便，並且載入速度也比較快。如果需要載入其他位置的資源，則應使用 `cc.loader.load` 方法。

##### 如何釋放掉資源

在 Cocos Creator 中，可以使用`cc.loader.release`方法來卸載資源。該方法的參數可以是資源的路徑（字符串）或資源對象本身。

```js
// 載入一張圖片資源
cc.loader.loadRes('textures/image', cc.SpriteFrame, function (err, spriteFrame) {
    if (err) {
        cc.error(err.message || err);
        return;
    }
    // 使用該資源
    var sprite = node.getComponent(cc.Sprite);
    sprite.spriteFrame = spriteFrame;

    // 釋放該資源
    cc.loader.release(spriteFrame);
});

```

在上面的範例中，首先使用 `cc.loader.loadRes` 方法載入了一個圖片資源，然後使用 `spriteFrame` 物件來更新一個 `cc.Sprite` 組件的圖片。最後，使用 `cc.loader.release` 方法來釋放該資源，以釋放記憶體。

{% note info%}

當釋放一個資源時，如果該資源正在被其他物件使用，則該資源不會真正地被釋放。只有當該資源不再被任何物件引用時，才會被真正地釋放。因此，在釋放資源之前，應該確保該資源已經不再被使用了。

{% endnote %}

##### 什麼是非同步載入資源?對寫程式碼有什麼影響?

在 Cocos Creator 中，資源載入是非同步進行的，也就是說當使用 `cc.loader.loadRes` 或 `cc.loader.load` 方法載入資源時，這些方法會立即返回，而不是等待資源載入完成。

這種非同步載入資源的方式在某種程度上提高了程式的效率和使用體驗。如果在資源載入期間程式阻塞，那麼用戶會感覺到應用程式變得緩慢或不穩定。而使用非同步載入資源的方式，可以在資源載入的同時繼續執行程式，從而提高應用程式的反應速度和流暢度。

寫程式碼時，需要注意非同步載入資源可能帶來的影響。由於資源載入是非同步進行的，因此在使用載入的資源之前，必須確保資源已經載入完成。否則可能會出現一些問題，例如資源尚未載入完成，但程式已經使用該資源，這樣會導致程式出錯或顯示錯誤的內容。為了避免這種情況，可以在資源載入完成後再使用資源，或者使用回調函數（Callback Function）在資源載入完成後執行相應的操作。

#### 如何編寫單體模式

##### 如何實現單體模式（Singleton）?

[Singleton Pattern 單體模式](https://medium.com/@ken556621/design-pattern-單體模式-5f00583778c4)

在 Cocos 中實現單體模式，可以使用靜態成員變量或函數來實現。

```js
var Singleton = cc.Class({
    statics: {
        instance: null,
        getInstance: function () {
            if (!this.instance) {
                this.instance = new Singleton();
            }
            return this.instance;
        }
    },
    
    ctor: function () {
        // 初始化代碼
    }
});

// 測試單體模式
var instance1 = Singleton.getInstance();
var instance2 = Singleton.getInstance();
console.log(instance1 === instance2); // true，instance1 和 instance2 是同一個對象

```

`Singleton` 是一個 Cocos 的 Class，它定義了一個靜態的 `instance` 成員變數和一個靜態的 `getInstance` 方法。當 `getInstance` 方法被調用時，它會檢查 `instance` 是否已經存在。如果不存在，就創建一個新的實體（instance），並將其賦值給 `instance`，然後返回這個實體（instance）。如果已經存在，就直接返回 `instance`。

由於 `instance` 成員變數是靜態的，外部程式碼可以通過 Class 名直接訪問它，因此可以保證只有一個實體（instance）被創建。

{% note info%}

在 Cocos 中，單體模式常常被用來管理全域資源，例如音效、音樂、遊戲資料等。在使用單體模式時，需要注意控制資源的生命週期，避免造成記憶體泄漏和性能問題。

{% endnote %}

##### 組件實體的單體如何保證全域唯一?

1. 全域節點
可以在場景的根節點上添加一個全域節點，將所有需要全域唯一的組件添加到這個節點上。在其他組件中需要訪問全域組件時，通過查找全域節點的方式來獲取該組件的實例。

```js
var GlobalNode = cc.Class({
    extends: cc.Component,

    statics: {
        instance: null,
    },

    onLoad: function () {
        GlobalNode.instance = this;
    },

    // 省略...
});

var MyComponent = cc.Class({
    extends: cc.Component,

    onLoad: function () {
        this.globalComponent = GlobalNode.instance.getComponent('GlobalComponent');
        // 訪問全域組件...
    },

    // 省略...
});
```

2. 全域變數
可以在全域範圍內定義一個變數來保存組件的實例，在需要訪問組件的地方通過讀取全域變數的方式來獲取組件的實體。

```js
var GlobalComponent = cc.Class({
    extends: cc.Component,
    
    statics: {
        instance: null,
    },

    onLoad: function () {
        GlobalComponent.instance = this;
    },

    // 省略...
});

var MyComponent = cc.Class({
    extends: cc.Component,

    onLoad: function () {
        this.globalComponent = GlobalComponent.instance;
        // 訪問全域組件...
    },

    // 省略其他代碼...
});

```

##### 如何實現常駐節點？不隨場景切換而消亡

要實現常駐節點，可以在任意場景中創建一個節點，並將其設置為常駐節點，這樣該節點及其子節點都不會隨場景切換而消失。

可以在 cocos creator 的編輯器中創建一個新的節點，然後在屬性檢查器中選擇「常駐節點」的屬性，或者通過編程方式在該節點的組件中設定其為常駐節點：

`cc.game.addPersistRootNode(node);`

其中，node 為要設置為常駐節點的節點對象。

這樣設置之後，該節點及其子節點就會一直存在於整個應用程式的生命週期中，不會隨場景的切換而消失。需要注意的是，常駐節點不會隨場景的切換而消失，但是如果應用程式退出，常駐節點還是會被釋放掉。

------

##### 列舉一下什麼情況下會用常駐節點?

1. 全域設置：如果需要在整個應用程式中共用一些設置或狀態，例如語言選擇、音樂開關等，可以使用常駐節點保存這些設置或狀態，以便在整個應用程式中共用。

2. 全域管理器：如果需要管理一些全域的資源，例如音效、動畫、物件池等，可以使用常駐節點作為管理器，以便在整個應用程式中方便地訪問和管理這些資源。

3. 資料緩存：如果需要在不同場景之間共用一些資料，例如遊戲進度、用戶設置等，可以使用常駐節點作為資料緩存，以便在不同場景之間共用這些資料。

#### Label 的原理與如何節約 Drawcall?

Cocos Creator Label 是一種用於顯示文本的組件，它允許使用者在遊戲中顯示文本。Cocos Creator Label 的原理是使用 BitmapFont 或 TTF 字體文件，將字符轉換為紋理圖，然後使用紋理圖顯示文本。

為了節約 Drawcall，可以採用以下幾種方法：

1. 合併 Label：將多個 Label 合併為一個 Label，可以減少 Drawcall。

2. 合併紋理圖：將多個紋理圖合併為一個紋理圖，可以減少 Drawcall。

3. 使用 Distance Field 字體：Distance Field 字體是一種可以無限放大而不失真的字體，它可以減少 Drawcall。

4. 合併 Batch 渲染：使用 Batch 渲染可以將多個 Sprite 合併成一個 Batch 渲染器，可以減少 Drawcall。

5. 使用 Atlas 紋理集：使用 Atlas 紋理集可以將多個紋理圖合併為一個紋理集，可以減少 Drawcall。

6. 減少 Label 的使用：如果可能的話，可以減少 Label 的使用，使用其他組件或圖片代替 Label，可以減少 Drawcall。

##### Label 底層是如何繪製的?

在 Cocos Creator 中，Label 組件使用 Canvas 模式來渲染文字，即在畫布上直接繪製文字。底層實現是通過使用 HTML5 Canvas API 實現的，包括填充文字、描邊文字、應用字體和樣式等操作。

當我們創建一個 Label 組件時，Cocos Creator 會自動創建一個 Canvas 節點，並將該節點作為 Label 組件的根節點，然後通過 Canvas API 在該節點上繪製我們指定的文本。由於使用了 Canvas 繪製，因此 Label 組件的渲染速度相對較快。

在 Label 組件中使用的字體文件是需要進行載入的，因此如果使用了較多的不同字體，會增加遊戲的包體大小。同時，為了避免在遊戲運行時動態載入字體文件，可以在遊戲啟動時預載入所有需要使用的字體文件，從而提高遊戲的載入速度。

##### 每個 label 一個 drawcall 如何優化

在 Cocos Creator 中，每個 Label 組件預設情況下都會獨占一個 Drawcall，這意味著如果場景中存在大量的 Label 組件，會對遊戲的性能造成不良影響。為了優化 Drawcall 數量，我們可以採用如下方法：

合併 Label：將相鄰的 Label 組件合併成一個組件，從而減少 Drawcall 數量。在實際開發中，我們可以使用 BMFont 工具將多個字體生成一張貼圖，然後將不同的文本內容顯示在不同的區域，從而達到合併 Label 的效果。

禁用 Label 組件的 outline 屬性：在 Label 組件中使用 outline 屬性會增加 Drawcall 數量，因此如果不需要使用該屬性，可以在組件屬性中將該屬性禁用，從而減少 Drawcall 數量。

使用 BitmapFont：BitmapFont 可以將所有字符合成到一張貼圖中，從而實現在一次 Drawcall 中渲染出所有字符的效果，這種方式可以有效地減少 Drawcall 數量。

使用 LabelAtlas：LabelAtlas 是 Cocos Creator 中專門用來顯示靜態文本的組件，它可以將所有字符合成到一張貼圖中，並使用指定的紋理坐標顯示字符，從而實現在一次 Drawcall 中渲染出所有字符的效果。需要注意的是，LabelAtlas 只能顯示靜態文本，不支持動態修改文本內容。

在實際開發中，需要盡可能地減少 Drawcall 數量，從而提高遊戲的性能。

##### 性能參數 drawcall 是什麼意思? 降低 drawcall 有什麼好處?

drawcall 是指 CPU 與 GPU 之間的一種通訊方式，用來告訴 GPU 什麼時候需要畫一個物體。簡單來說，drawcall 的數量就代表著畫面中需要執行多少次繪製命令，一般來說 drawcall 數量越多，性能就會越差。

降低 drawcall 數量的好處有以下幾點：

1. 提升遊戲的性能和流暢度。
2. 減少 GPU 與 CPU 之間的通信次數，減少 CPU 的負擔。
3. 提升遊戲的可玩性，減少因為性能問題而出現的卡頓現象。

降低 drawcall 數量是優化遊戲性能的一個重要手段。在開發過程中，需要注意避免出現不必要的 drawcall，比如合併多個 sprite 為一個大的 sprite，減少 label 的使用等。

##### 2D遊戲開發降低 drawcall 要注意那些事情?

在2D遊戲開發中，降低 drawcall 是一個非常重要的優化手段，可以提高遊戲的性能

1. 合批渲染：盡量減少不同的貼圖和材質的使用，將相同的材質合併在一起渲染，可以減少 drawcall 的數量。

2. 使用 SpriteFrame：如果需要渲染多個相同的圖片，可以使用 SpriteFrame 來實現，這樣可以減少 drawcall 的數量。

3. 合理使用裁剪：使用裁剪可以避免一些不必要的渲染，也可以減少 drawcall 的數量。

4. 適當使用 SpriteAtlas：將圖片打包成 SpriteAtlas 可以減少材質切換的次數，從而減少 drawcall 的數量。

5. 適當使用動態合批：當需要動態渲染大量相同材質的物體時，可以使用動態合批技術，將相同材質的物體合併成一個物體，減少 drawcall 的數量。

6. 減少場景中的物體數量：減少場景中的物體數量也可以減少 drawcall 的數量。

##### 如何組織場景節點的順序，來降低drawcall ，有哪些注意的點?

1. 使用同一個圖集：將相同材質的 2D 圖片打包到同一個圖集中，然後在 Cocos Creator 中將它們拖到同一個 SpriteFrame 中，最後在 Scene Editor 中將它們拖到同一個 Sprite 節點下，這樣可以減少drawcall 的數量。

2. 使用 Sprite 合批：將多個 Sprite 節點合併到同一個Sprite節點下，這樣可以減少 drawcall 的數量。但是需要注意的是，合併的 Sprite 節點的材質必須相同，且不能有太多節點被合併，否則會造成過度的延遲。

3. 調整節點的層級：在 Scene Editor 中，將需要在同一個 drawcall 中的 Sprite 節點放到同一個層級下，這樣可以減少 drawcall 的數量。但是需要注意的是，如果節點的層級太多，也會對性能造成影響。

4. 適當使用 Mask：如果需要顯示的圖片只有一部分，可以使用 Mask 來限制顯示範圍，這樣可以減少 drawcall 的數量。

5. 適當使用 CCRenderer：在需要特別控制渲染順序或者需要使用深度測試的場景中，可以使用CCRenderer 來控制渲染順序，這樣可以減少 drawcall 的數量。

注意事項：

1. 不要過度使用合批和層級調整，否則會對性能造成影響。

2. 確保圖片的尺寸盡量小，這樣可以減少 GPU 的負擔。

3. 使用合理的Mask，避免過多的 Mask 導致性能下降。

4. 適當地控制渲染順序，避免過多的深度測試。

##### 背包系統文字過多導致 fps 降低如何優化

2. 減少 label 的數量：如果是過多的 label 導致 FPS 降低，可以考慮合併多個 label，減少 label 的數量。比如，將顯示相同類型的文本的 label 合併成一個，然後在文本的前面添加類型標籤。

3. 使用位圖字體渲染 UI：如果 UI 中也有大量的文本，可以考慮使用位圖字體來渲染 UI，進一步減少 GPU 和 CPU 的運算量。

4. 使用引擎自帶的渲染擴展功能：Cocos Creator引擎提供了一些渲染擴展功能，如LabelOutline、LabelShadow等，可以用來優化label的渲染效果。

5. 減少節點數量：如果節點數量過多也會影響 FPS，可以考慮減少節點的數量，比如通過製作背景圖集、合併多個圖片等方式減少圖片的數量。

#### WebSocket 與 Socket.IO 分別是什麼?

Cocos Creator WebSocket和Socket.IO都是用於實現網路通訊的工具，但是它們有一些不同之處。

WebSocket 是 HTML5 中新增的一個瀏覽器原生的通訊協議，它基於 TCP 協議，並提供了全雙工通信能力。在 Cocos Creator 中，可以使用 cc.WebSocket Class 進行 WebSocket 通信，它可以發送和接收文字或二進位資料，也可以設定回調函數（callback function）來處理接收到的消息。

Socket.IO 則是一個建立在 WebSocket 之上的高級框架，它提供了更高層次的抽象和功能，比如及時的事件傳輸和房間管理等。在 Cocos Creator 中，可以使用第三方函式庫 socket.io-client 進行Socket.IO 通訊，它提供了類似於 WebSocket 的 API ，同時還支持 Socket.IO 專有的事件機制。

##### Websocket 是什麼?為什麼 cocos creatot 用 websocket?

WebSocket 是一種網路通訊協定，可以在單個 TCP 連接上提供全雙工的通訊通道，它可以用於在網頁和網頁服務器之間進行即時通訊和資料傳輸。

Cocos Creator使用WebSocket是因為WebSocket具有以下優點：

雙向通訊：WebSocket 是一個基於 TCP 的協議，支援客戶端和伺服器之間的雙向通訊，可以實現即時性的資料交換，例如即時遊戲中的玩家互動、聊天等功能。

較小的資料包：WebSocket 協議比 HTTP 協議傳輸的資料包更小，因為 WebSocket 在傳輸資料時沒有 HTTP 協議那樣的請求和響應頭（header），只傳輸資料內容。

基於事件驅動：WebSocket 協議是基於事件驅動的，伺服器可以推送資料到客戶端，而不需要客戶端主動向伺服器發送請求。

WebSocket 具有快速、高效、即時、可靠的特點，非常適合在遊戲開發中使用。

##### Socket.IO 是什麼? 如何使用?

Socket.IO 是一個開源的JavaScript Libary，用於實現 WebSocket 通訊，並提供了即時的雙向通訊和可靠的傳輸機制。它可以在瀏覽器和伺服器之間建立持久性連接，並支援各種事件和自定義協議。

使用 Socket.IO，您可以輕鬆地在 Web 應用程式中實現即時通訊，例如聊天應用程式、多人遊戲、線上編輯器等。

在 Cocos Creator 中使用 Socket.IO，您需要首先在伺服器端安裝 Socket.IO ，然後在客戶端程式碼中使用 Socket.IO 客戶端 Libary。可以使用以下程式碼示範來連接 Socket.IO 伺服器：

```js
const io = require('socket.io-client');

// connect to server
const socket = io('http://localhost:3000');

// listen for 'connect' event
socket.on('connect', () => {
  console.log('connected to server');
});

// listen for 'message' event
socket.on('message', (data) => {
  console.log(`received message: ${data}`);
});

// send 'message' event to server
socket.emit('message', 'hello, server');

```

上述程式碼中，我們使用 Socket.IO 客戶端庫連接到伺服器，並註冊了 connect 事件和 message 事件的監聽器。當與伺服器建立連接時，connect 事件會觸發，當從伺服器收到消息時，message 事件會觸發。我們還使用 emit 方法將 message 事件發送到伺服器。

##### 項目中到底用 websocket 和 socket.io 你怎麼看

在使用 WebSocket 和 Socket.IO 之前，需要先了解它們各自的特點和適用情況。WebSocket 是一個 HTML5 的標準，可以在 Web 端和伺服器端之間建立一個持久性的連接通道，以實現全雙工的通訊。WebSocket 可以在客戶端與伺服器之間實現高效的、即時的通信，非常適合用於遊戲伺服器和客戶端之間的通訊。

而 Socket.IO 是一個封裝了 WebSocket 的函式庫，支持即時雙向通訊，同時提供了更多的功能和語法糖，例如實現房間和命名空間，消息的序列化和反序列化等等。Socket.IO 適合用於需要在伺服器和客戶端之間實現複雜的通信場景，例如聊天室、多人遊戲等等。

因此，當需要實現簡單的、高效的通信時，建議使用 WebSocket。而當需要實現複雜的通訊場景時，Socket.IO 可能更加適合。

在 Cocos Creator 中，可以通過引入 WebSocket 或 Socket.IO 的函式庫來實現相應的通訊功能。

##### 微信小遊戲是否使用 UDP

微信小遊戲目前不支援使用 UDP 協議，只支援使用 WebSocket 和 HTTP 協議。如果需要使用 UDP 協議，可以考慮使用其他平台進行開發。

##### WSS 是什麼? websocket 如何支援 WSS?

WSS代表"WebSocket Secure"，是一種透過 TLS / SSL 加密協議保護 WebSocket 協議的安全版本。 它使用了TLS / SSL協議對 WebSocket 數據進行加密，從而提供了更高的安全性，避免了被攔截和窺視的風險。WSS 與 WebSocket 協議相比，可以更好地保護敏感訊息與用戶隱私。

為了支援WSS，WebSocket客戶端需要使用 `https:`協議來連接服務器。在 Cocos Creator 中，可以使用 `new WebSocket('wss://yourserver.com')` 來創建一個 WSS 連接。

##### Websocket 發送資料有哪些模式? 如何設置

Websocket 發送數據有兩種模式：

文字模式（Text mode）：發送的數據是一段純字串。

二進位模式（Binary mode）：發送的資料是二進位資料流。

在 Cocos Creator 中，可以使用 WebSocket 的 send 方法來發送資料，該方法接收一個參數，表示要發送的資料，參數類型可以是字串或 ArrayBuffer。

下面是一個發送文本和二進位資料的範例：

```js

// 創建一個 WebSocket 對象
var ws = new WebSocket("ws://localhost:8080");

// 發送文本數據
ws.send("hello world");

// 發送二進制數據
var arrayBuffer = new Uint8Array([1, 2, 3]).buffer;
ws.send(arrayBuffer);

```

在 Cocos Creator 中，WebSocket 物件的 binaryType 屬性可以用來設置接收二進位資料時的資料類型，可以設置為 "arraybuffer" 或 "blob"，預設為 "blob"。例如，可以使用以下程式碼來設定

```js
binaryType：ws.binaryType = "arraybuffer";
```

這樣在接收二進位資料時，WebSocket 物件就會返回一個 ArrayBuffer 物件。
