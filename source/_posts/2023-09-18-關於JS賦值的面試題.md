最近一些面試題都有考到相關的試題，很顯然我都答不出來被電的咪咪卯卯
這篇主要探討一些簡單的問題
ex:
```js
let a = '胖胖'
let b = '胖胖'
a===b //答案為?
let c = [1,2,3]
let d = [1,2,3]
c===d //答案為?

```
<!--more-->

透過 Huli 所分享[關於 JS 變數](https://hulitw.medium.com/variable-and-frontdesk-a53a0440af3c)的一個簡單舉例來學習。

拜讀後可以得知以下結論

## **非物件**情況下
當一個變數在 JS 中被賦值，且不為物件(陣列)的情況下，每個變數都是獨立個體

```js
let a = 'Apple'
let b = 'Bee'
a===b //false
//當 b 賦值 a 時則舊有的資料被取代
b=a
a===b //true
```

## **物件**情況下
則為標記指向，c和d雖然資料相同，但指向不同

```js
let c=[1,2,3]
let d=[1,2,3]
c===d //false
```

那如果給 d 賦值 c 呢?

```js
let c=[1,2,3]
let d=[1,2,3]
d=c
d===c//true
```
因為 d 指向被修改為 c ， 此時 d 與 c 都是指向都一個物件
所以這時再修改 d 時 c 也會跟著被修改

```js
let c=[1,2,3]
let d=[1,2,3]
d=c
d===c//true
d.push(4)
console.log(c) //[1,2,3,4]

```

如果當再次被重新賦值，其舊有的指向就會被拋棄，被新的指向所取代，就算內容一樣
不管再怎麼修改其資料都跟另一個變數指向完全沒有關係


## 結論
這邊會簡單列幾個我碰到的面試考題基本上跟上面的差不多
我會比較有疑問的可能是為什麼可以直接`console.log({}==={})`而不會噴 undifiend

詢問了一下 chatGPT 獲得的答案是:
當我們在使用物件的那個時候，其實已經在記憶體自動創建一個指向的物件，所以才不會噴錯

```
let a = 123
let b = 123
a===b //true
{}==={} //false
[]===[] //false

```

總之以上就是我對於賦值的理解

只要先看是否為物件，再判斷是否重新被賦值或只是更改指向的問題就很好理解了。
