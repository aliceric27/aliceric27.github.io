# 面試問題

## 什麼是 hoisting

屬於 var 宣告優先序問題

舉簡單例子

```js
console.log(a) //undifiend
var a = 5
```

此例子中可以想像程式這樣執行

```js
var a
console.log(a) //undifiend
a = 5
```

搭配 function 型態的變種則為

```js
function test(v){
    console.log(v)
    var v = 3
}
test(5) //5

```

因為在 function 呼叫時會初始化變數，可以想像程式是這樣執行

```js
function test(v){
    var v = 5
    var v
    console.log(v)
    v = 3
}
test(5) //5

```

**可是**我在 log 前不是已經將 v 賦值為`undifiend`了嘛!?
可以簡單舉例

```js
var a = 5
var a 
console.log(a) //5
```

最後還有一個小節 ， 當作用域都在全域下時應該怎麼執行

```js
console.log(a) //fn(a)
var a
function a(){}
```

此時 log 出來的 a 會是`function`而不是`undefined`。

基本重點整理

1. 變數宣告和函式 (fn) 宣告都會提升
2. 只有宣告**會**提升，賦值**不會**提升
3. 函式要考慮傳進來的參數也會被初始化

### let 與 var 綜合題型

這也是我這次面試栽了的地方，因為現在語法多為 let 和 const 宣告，比較少會碰到混合宣告的方式。

題目如下

```js
var a = 10
function test(){
  console.log(a)
  let a
}
test()
```

我當下會認為不就是 10 嘛，var在最外面宣告為 10 ，那全域變數 a 就是`10`。

答案是`ReferenceError: a is not defined`。

面試當下心態直接炸裂，開始慌亂回答了，我難過QQ。

上方標題連結到我的虛擬線上導師 Huli 所撰寫的技術文章
篇幅較長就不一一節錄，主要的重點是按規格書的規則下去執行

拆解以下程式碼其實可以分為兩步驟

```js
function test(v){
  console.log(v)
  var v = 3
}
test(10)

```
在 function 執行期間可以分為兩個階段，首先會先初始化相關參數
使用文章內的 VO 概念建立一個初始化物件

```js
{
    v:10
}
```

然後才開始執行 function ，此時的 v 為`10`當然會印出的就是`10`。

將題目轉換一下，套用剛才的方式很簡單的就可以知道印出的分別為`10`和`3`。
因為在下一行 v 已經被賦值為`3`。

```js
function test(v){
  console.log(v)
  var v = 3
  conosle.log(v)
}
test(10)

```

這樣了解規則後變種型態其實理解也不會很困難

```js
var v=4
function test(v){
    let a=2;
    console.log(v)//10
    console.log(a)
    let v = 3
}
test(10)

```
首先會先將需要的資料宣告完成這樣理解作用域則為

```js
global{
    v:undifiend
}
test{
    v:undifiend
}
```

## CSS 相關基本問題


面是有詢問到 CSS 比較少用到的選擇器，其實滿基本的，但因為實務上少用到所以忘得差不多
趁這個機會來複習一下

### CSS 基本選擇器
首先是基本的 id、class 選擇以及同時符合的規則

```html
<div id="a" class="aclass">apple</div>
<div id="b" class="bclass">Bee</div>
<div id="c" class="bclass">Cat</div>
<div class="dog">
  <div>Dog</div>
</div>
```
```css
/* 改變所有的元素 */
*{
  background-color:grey;
}
/* ID選擇器 */
#a{
color:red;
}
/* class選擇器 */
.bclass{
color:green
}
/* 同時符合兩個條件，不需要空格 */
#c.bclass{
  color:yellow
}
/* 空格為選擇子項可以是id、class、標籤 */
.dog div{
  font-size:20px;
}

/* 下一層選擇器，這個和空格不同的是只會影響第一個子層的元素 */
.dog > div{
  color:blue;
}

```

上面是比較基本的選擇器，也覆蓋 8 成以上使用情境
接下來會複習比較少用到的

### CSS + 和 ~ 同層選擇器

這兩個選擇器只有在同層下才會有作用


```html
<p class='a'>apple</p>
<p class='a'>bee</p>
<p class='a'>
  cat
<span class='d'>cat</span>
</p>
<p class='a'>dog</p>
```

```css
/* 只有.a class 後方所有的.a都會變色*/
.a ~ .a{
color:red;
}

```

如果將 class 改為`+`則會選擇在設定條件的同一層的**下一個**符合的元素來做對應改變。
如果同層中間有其他元素中斷了這個規則，那他就不會被選擇到

```html
<p class='a'>apple</p>
<p class='a'>bee</p>
<p class='a'>
<span class='a'>dog</span>
</p>
<div>test</div>
<p class='a'>pig</p>
```

```css
/* 只有 bee 和 dog 會改變顏色*/
.a + .a{
  color:yellowgreen;
}
```

### CSS權重

基本上 CSS 權重區分如下

`!>id>class>tag`

CSS 會將所有權重下去做計算

套用的規則會根據 CSS 設定的權重下去評分，如過評分都一樣則 CSS 樣式會由後面的設定覆蓋前面的設定

以下是簡單分數評斷

沒有任何定義的權重分是這個樣子
`ID 0 , Class 0 , tag 0`
假設最前方 ID 為 1 則後方都不用比較，權重一定最高
`ID 1 , Class 0 , tag 0`

可以想像一個 ID 是百元鈔、一個 class 是十元硬幣、tag為一元硬幣這樣下去比較

舉一個簡單的例子，假設以下都是定義同一個物件最後顏色會是 blue
因為 ID 權重為 `1`，下面的三個 class 加起來為`3`還是被 ID 權重蓋過去

用剛剛的貨幣理論， 100 元 > 30 元這種感覺

```css

#apple{
color:blue
}

.appple .bee .cat{
  color:yellow
}

```

相反的如果權重都相同，就會用後面設定的元素覆蓋前面設定的元素
下面輸出會因為權重相同，所以最後結果為`yellow`

```css

.apple > .bee > .cat{
color:blue
}

.appple .bee .cat{
  color:yellow
}

```

最後有個例外千元鈔，那就是`inline-style`
這個權重是蓋在ID上方的，新增完這個權重則計算方式變為下表
`inline-style > ID > class > tag`

```html
<p style='color:red'>test</p>
```

最最最後有個超級萬元鈔，那就是`!important`語法
這個萬元鈔殺手鐧可以覆蓋所有權重

```css
.apple{
  color:red !important
}
#apple{
  color:blue
}
```

就算上面請了百元鈔 ID 大哥，和位置放後也敵不過`important`大哥，結果為會顯示`紅色`
最後權重區分權重表如下
{% note default %}

! > inline-style > id > class > tag

{% endnote %}
