## Why - 為什麼會進來上課

一言以蔽之：**因為我沒有一技之長**

先說說我的背景吧，我畢業於樹德科技大學-資訊管理系

大學四年可以拿來說的出來的成就只有參加過某雪的射擊遊戲的實體電競比賽，青春 R..🥲

現在滿後悔那時學校上課沒有乖乖把程式語言修好，整個大學都在玩，甚至一路玩到我進資策會的前幾個月

是的，你沒有看錯，我非常喜歡遊戲，不過到了一個年紀後，開始變的不這麼愛玩

~~一定不是買不到 PS5，還是這幾年出的遊戲評價太差~~

當時不認真寫程式也是因為程式寫起來和遊戲比一點都不有趣，當年唯一的畢業專題甚至是拿 Wordpress 套用現成套件所做的購物網站，在當時還得獎，我其實那時一直不能理解，比我們有趣的專題很多，為什麼只有我們得獎？

> 直到現在我才知道，不依靠**套件**寫出購物網站是多難的事情

## What - 上課都學了些什麼

我報名了全端工程師課程，在這之前我有在網路上自學 1~2 個月左右的時間，對我來說學習成效其實沒有特別好

特別容易分心，且做的作品與學習的路線沒有帶入門與方向性，加上課程有政府的輔助，不上白不上

我從 2022 年 9 月底到 2022 年底完成所有課程，可以和讀者分享我的課程與心得

### 9 月~10 月 前端課程

`HTML` 5 天 - 講解 HTML 基本元素語法語用法、重點著重在 **Form 表單**的用法上，沒什麼難度的快樂第一個禮拜 😇

`CSS` 5 天 - 講解 CSS 基本用法，選擇器、文字處理、圖片處理、Box Model、Animation、Position、Display

這個禮拜開始慢慢加速，滿意外的是沒有教 Flex 排版用法，這部份我再做第一個專題網站時自己上網補習的，可能是授課時間關係所以沒辦法教到這麼多我覺得滿可惜的，不過教的基本的屬性量也滿多的，如果有作筆記，常常回去翻，最常翻的筆記是**選擇器**覺得最重要。

`Boostrap` 3 天 - 個人覺得最浪費時數的課程，卡到國慶連假，教學 Boostrap 的用法從這時候開始老師會強調，

查看文件的重要性，覺得這個課程學習 Boostrap 用法是其次，不過我覺得這個課程可以壓縮到 1 天解決，剩下的時間

給 JS 多 1.2 天時間授課比較合理。

`Javascript` 6 天 - 這 6 天我真的回想不起來我到底學了什麼，因為一次塞太多東西了，開頭教的記得第一天就學習模板字串符，然後是作用域 `let`、`const` ES6 的新語法..剩下的我都不太記得。

因為教的太快所以也不確定破碎的 JS 學習到底是從哪邊學的，大部分是照我前一兩個月的自學印象搭配網路上豐富資源跟著學慢慢銜接上，看筆記總共教了作用域、字串、陣列用法、function、物件導向、事件處理、canvas 繪製。

因為 JS 的老師教的非常快，所以一定要自己額外找許多資料自學，尤其是物件導向的部分，JS 授課完後離訓人數達到最高峰。

> 我認為有心要轉職學習，再來報名會比較好，至少先自學 JS 或其他程式語言看自己的接受度如何。

---

### 10~11 月 Jquery 與後端課程

`JQuery` 3 天 - 主要複習 Javascript 的語法，與常用的 JQuery 語法，還有簡單查看文檔使用方式

`SQL` & `MySQL` 8 天 - SQL 我自己也沒學的很好，個人感覺 SQL 課程有點太多，其實後期做專題時發現最簡單的 CRUD 指令一套就可以做非常多事情了，我們 8 天還教了如何在 MySQL 內寫程式碼跑迴圈，雖然有老師說我們班比較偏後端，不過 SQL 教成這樣子..實在不予置評，學完後現在還是只會最簡單的 CRUD 指令。

`期中專題` 6 天 - 這 6 天就是給我們完成自己一個專題用的，用前面所學來實作，時間其實非常充裕，最後完成的作品如下

[GitHub](https://aliceric27.github.io/littleweb/)

`PHP` 7 天 - 這 7 天收穫非常滿，因為有 Javascript 的基礎，所以在吸收與理解上非常快，除了 PHP 最詭異的物件選取方式是用"->"表示外，剩下的都很好理解，老師教課也非常熟稔，課堂上一句廢話都沒有，是用實作小作品的方式教學，代碼也會同步傳到 GitHub 讓同學課後學習，雖然現在很多都還老師了，不過 PHP 學習的體驗算很愉快，老師主要是教 JAVA 。

`Laravel` 5 天 - 這 5 天的 Laravel 學習狀況個人認為成效滿不彰的，因為第一次接觸 **MVC** 架構，花了許多時間在了解基本架構原理，以及環境的設定上，因為許多人的系統都不一致導致很多人花了許多的時間在環境建置上卡關，為此我也寫了一份 MAC 上給學弟妹們參考的環境架設文件

[MAC - Laravel 環境設定](https://hackmd.io/@aliceric27/rJoCmWSPo)

Laravel 框架最後也學習的比較凌亂，期待未來有機會在專案上使用時，會再回去加強複習。

---

### 12 月~課程結束

`RestFul` 3 天 - 主要講述前端接 API 並處理傳回資料的方式，課程受益良多，也在這時重新複習了前後端的基本溝通概念。

`React.js` 6 天 - React 框架放在最後才學習，一開始學習的是比較傳統的 Class 方式撰寫，讓我再次回去複習了許多 Javascript 的基礎與 ES6 的各種新特性，因為課程放到最後，也很快接下來做期末專題有用到大量的前端框架特性，所以最後學習了各種 React 的 Hook function ，也改採用新的寫法來製作期末專題，所以 React 框架部分摸的會比後端更加熟練。

`其他課程` 若干天 - 其中還有許多課程像是`系統規劃`或是`系統分析`，分散在課程與課程之間，主要教我們專案的製作與基本專案管理，還有 Git 的基本使用，雖然在製作第一個專題的時候我已經自己學會了。

---

課程到這邊就完整結束了，雖然僅有短短的 3 個多月，不過比較難以置信的是在這 3 個月內學習了很多前、後端基礎，這樣的學習速度比起我自己在家摸了 1.2 個月自學還比較有效果，因為有同屆的同學互相學習與提問，會讓自己比較有動力去學習，畢竟不是只有自己卡在這個難點，有些難點需要大家一起尋找並歸納出來解決方案。

**以下是我的一些小建議給予參考**

- 如果是正在觀望的轉職者們 - 我會建議從自學起步，查找 Youtube、Google 上許多參考文章，精確學習使用關鍵字搜尋你要的東西是你目前最需要的技能，一定要將這個技能練到爐火純青，對於未來找資料會非常有幫助！

- 如果是正在學習或資策會學弟妹們 - 如果覺得現在學習的東西很困難，或是非常有障礙，很建議可以與同學討論或找尋更多不同人的解釋或書籍，或著可以選擇**放棄**，放棄不是要你完全失去學習，建議可以先用自己可以理解的方式嘗試解決問題，有時候過段時間就莫名其妙會理解當初自己卡住的問題點，而且常常會這樣！

---

## 課後結語

在學習時，學的愈多才會認知到自己會的東西愈少，以前沒有壓力的情況下在學習一樣事物的心態上會不夠認真與上心，來 3 個月資策會後，對於學習的心態有一定的改變，也期許未來的自己可以保持在這樣的學習心態上工作或是做任何事情。

PS:真的沒有附便當，很建議自己處理午餐，因為上課地點附近很多雷店在踩點時請慎重選擇！！

這篇文章獻給所有資策會未來的學弟妹們、與徬徨無助的轉職者，也告別 3 個月前渾渾噩噩的自己。
