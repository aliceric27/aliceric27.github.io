* 想讓 Vue 工作就必須創建（new）一個 Vue 物件
* HTML 容器內仍然遵循 HTML 規範，只是混入一些 Vue 的語法

* HTML 中的 `{{}}`內可以寫入所有 Js 表達式
* 一旦 Vue 物件的 data 發生改變，HTML 容器內使用的數值會隨時自動更新

### [Vue 模板語法](https://peterhpchen.github.io/VuejsQuest/basic/05_Mustache.html#%E6%A8%A1%E6%9D%BF)

#### 插值語法

功能：用於解析 Vue 物件 data 的設定值

寫法：{{xxxx}} ，xxxx 為 Js 表達式，可以直接讀取到 data 的所有屬性

#### 指令語法

功能：用於解析標籤（包括：標籤屬性、標籤體內容、綁定事件.....）。

 舉例：`v-bind:href="xxx"` 或簡寫為 `:href="xxx"`，xxx同樣要寫 Js 表達式，

且可以直接讀取到 data 中的所有屬性。

{% note info%}

Vue中有很多的指令，且形式都是：v-????，此處只是拿v-bind舉個例子。

{% endnote %}

### [資料綁定](https://peterhpchen.github.io/VuejsQuest/basic/21_ComponentData.html#%E7%88%B6%E7%B5%84%E4%BB%B6%E4%BD%BF%E7%94%A8%E5%B1%AC%E6%80%A7%E5%82%B3%E8%B3%87%E6%96%99%E7%B5%A6%E5%AD%90%E7%B5%84%E4%BB%B6)

Vue中有2種資料綁定的方式：

#### 單向綁定 (v-bind)：資料只能從data流向頁面

#### 雙向綁定 (v-model)：資料不僅能從data流向頁面，還可以從頁面流向 data

{% note info%}

* 雙向綁定一般都應用在表單類元素上（如：input、select等）

* v-model:value 可以簡寫為 v-model，因為 v-model 預設收集的就是value值。

{% endnote %}

### [data 與 el 的2種寫法](https://github.com/aliceric27/vue.pratice/blob/main/04_el%E4%B8%8Edata%E7%9A%84%E4%B8%A4%E7%A7%8D%E5%86%99%E6%B3%95/el%E4%B8%8Edata%E7%9A%84%E4%B8%A4%E7%A7%8D%E5%86%99%E6%B3%95.html)

#### el

1. new Vue 時候配置 el 屬性。

```js
const v = new Vue({
el:'#root', //第一種寫法
data:{
name:'尚硅谷'
 }
})
```

2. 先 new Vue，隨後再通過`vm.$mount('#root')`指定 el 的值。

```js
const v = new Vue({
data:{
name:'尚硅谷'
 }
})
v.$mount('#root')
```

#### data

1. 對象式

```js
 //data的第一種寫法：對象式
    data:{
    name:'尚硅谷'
    } 
```

2. 函數式

```js
data(){
console.log('@@@',this) //此處的this是Vue實例物件
return{
  name:'尚硅谷'
 }
}
```

​    如何選擇：目前哪種寫法都可以，以後學習到組件時，data 必須使用函數式，否則會報錯。

   {% note warning%}

   由Vue管理的函數，一定不要寫箭頭函數，一旦寫了箭頭函數，this就不再是Vue實例了。

   {% endnote %}

### [資料代理](https://github.com/aliceric27/vue.pratice/tree/main/06_%E6%95%B0%E6%8D%AE%E4%BB%A3%E7%90%86)

學習 `Object.defineProperty()`這個內建函數，

這個函數需要傳入3個參數，物件本身、需要定義的值(字串)、設定參數

其中最重要的屬性為 `get()`、`set()`

`get()`替物件參數設定一個 function，當訪問這個參數時隨即 return 對應 function 回傳值

`set()`當物件參數改變數值時，調用 `set()`所設定的 function

```js
let number = 18
 let person = {
 name:'張三',
 sex:'男',
 }

Object.defineProperty(person,'age',{
 // value:18,
 // enumerable:true, //控制屬性是否可以遍歷，預設值是false
 // writable:true, //控制屬性是否可以被修改，預設值是false
 // configurable:true //控制屬性是否可以被刪除，預設值是false

 //當有人讀取person的age屬性時，get函數(getter)就會被調用，且返回值就是age的值
 get(){
 console.log('有人讀取age屬性了')
 return number
 },

 //當有人修改person的age屬性時，set函數(setter)就會被調用，且會收到修改的具體值
 set(value){
 console.log('有人修改了age屬性，且值是',value)
 number = value
 }

})

// console.log(Object.keys(person))

console.log(person)
```

Vue 當中也用到了這個特性，當我們 new Vue 後，並賦值給一個變數，則會將 Vue 內的 data 物件傳到這個變數中，所以兩邊數值是一模一樣的，只是傳入後 Vue 會幫我們加上 getter 與 setter 等方法，所以我們才可以使用模板函數直接讀取變數 {{xxx}}。

{% note info%}

存到變數中的 data 會以 _data 命名，所以模板輸入 {{_data.xxxx }} ，也是可以讀取到該值

{% endnote %}

### [事件處理](https://peterhpchen.github.io/VuejsQuest/basic/16_Event.html)

{% note primary%}

* 使用v-on:xxx 或 @xxx 綁定事件，其中xxx是事件名
* 事件的回調需要設定在 methods 物件中，最終會在 vm 上
* methods 中設定的函數，不要用箭頭函數！否則 this 就不是 vm 了
* methods中設定的函數，都是被 Vue 所管理的函數，this 的指向是 vm 或組件實例物件
* @click="demo" 和 @click="demo($event)" 效果一致，但後者可以傳入參數
{% endnote %}

#### 事件基本用法

使用`v-on`綁定事件，並填入調用方法名稱(字串)

事件綁定縮寫寫法為`@`

預設調用方法會傳入`event`事件(如果沒有傳入任何參數)

如果想要傳入參數則加上括號並傳入指定參數，想要保留預設 event 事件則需要傳入 `$event`

```html
<button v-on:click="showInfo">點我顯示訊息</button>

<button @click="showInfo1">點我顯示訊息1(不傳參數)</button>

<button @click="showInfo2($event,66)">點我顯示訊息2(傳參數)</button>
```

方法於 Vue 物件中的寫法：

需寫在 Vue 實例內的 methods 中，當 HTML 觸發方法時，Vue 會直接調用對應方法

```js
const vm = new Vue({
el:'#root',
data:{
name:'尚硅谷',
 },
methods:{
  showInfo1(event){
  // console.log(event.target.innerText)
  // console.log(this) //此处的this是vm
  alert('同学你好！')
  },
  showInfo2(event,number){
  console.log(event,number)
  // console.log(event.target.innerText)
  // console.log(this) //此处的this是vm
  alert('同学你好！！')
  }
 }
})
```

#### [事件修飾符號](https://peterhpchen.github.io/VuejsQuest/basic/17_EventModifier.html)

修飾符依照功能可以大概分為下面兩類：

事件修飾符: 事件相關的處理或設定。

按鍵修飾符: 鍵盤或是按鈕的事件中觸發的按鍵設定。

{% note primary%}
Vue中的事件修飾符：
    1. prevent：阻止預設事件（常用）
    2. stop：阻止事件冒泡（常用）
    3. once：事件只觸發一次（常用）
    4. capture：使用事件的捕獲模式
    5. self：只有 event.target 是當前操作的元素時才觸發事件
    6. passive：事件的預設行為立即執行，無需等待事件回調執行完畢
{% endnote %}

```html

<!-- 阻止預設事件（常用） -->
<a href="http://www.atguigu.com" @click.prevent="showInfo">點我提示信息</a>

<!-- 阻止事件冒泡（常用） -->
<div class="demo1" @click="showInfo">
<button @click.stop="showInfo">點我提示信息</button>

<!-- 修飾符可以連續寫 -->

<a href="http://www.atguigu.com" @click.prevent.stop="showInfo">點我提示信息</a>
</div>

<!-- 事件只觸發一次（常用） -->
<button @click.once="showInfo">點我提示信息</button>

```

{% note primary%}
Vue中常用的按鍵別名：
    1. 回車 => enter
    2. 刪除 => delete (捕獲“刪除”和“退格”鍵)
    3. 退出 => esc
    4. 空格 => space
    5. 換行 => tab (特殊，必須配合keydown去使用)
    6. 上 => up
    7. 下 => down
    8 . 左 => left
    9. 右 => right
{% endnote %}
基本用法

```html
<input type="text" placeholder="按下回車提示輸入" @keydown.enter="showInfo">
```

Vue未提供別名的按鍵，可以使用按鍵原始的 key 值去綁定，但注意要轉為 kebab-case（烤肉串命名）

```html
<input type="text" placeholder="按下回車提示輸入" @keydown.caps-lock="showInfo">
```

系統修飾鍵（用法特殊）：ctrl、alt、shift、meta

(1).配合 keyup 使用：按下修飾鍵的同時，再按下其他鍵，隨後釋放其他鍵，事件才被觸發。
(2).配合 keydown 使用：正常觸發事件。

```html
<!--需要同時按住 ctrl+y 才會觸發-->

<input type="text" placeholder="按下回車提示輸入" @keyup.ctrl.y="showInfo">

<!--按下則觸發-->

<input type="text" placeholder="按下回車提示輸入" @keydown.ctrl="showInfo">

```

也可以使用 keyCode 去指定具體的按鍵（不推薦）

```html
<input type="text" placeholder="按下回車提示輸入" @keydown.13="showInfo">
```

{% note danger%}

keyCode 已漸漸被瀏覽器棄用，且考量鍵盤規格不同不建議使用

{% endnote %}

Vue.config.keyCodes.自定義鍵名 = 鍵碼，可以去定製按鍵別名

```html

<input type="text" placeholder="按下回車提示輸入" @keydown.huiche="showInfo">

<script type="text/javascript">
Vue.config.productionTip = false //阻止 vue 在啟動時生成生產提示。
Vue.config.keyCodes.huiche = 13 //定義了一個別名按鍵

...
...
```

### [計算屬性](https://peterhpchen.github.io/VuejsQuest/basic/07_Computed.html)

一般來說如果要實現一個變數為兩個變數相加的字串可以用三種方式達成

1. 直接使用插值語法直接輸出

```html
<div id="root">
  姓：<input type="text" v-model="firstName"> <br/><br/>
  名：<input type="text" v-model="lastName"> <br/><br/>
  全名：<span>{{firstName}}-{{lastName}}</span>
</div>
```

2. 使用 method 方法 return 一個值

缺點：如果需要多個 fullName 每次都要呼叫一次方法導致效能低落

```html
<div id="root">
  姓：<input type="text" v-model="firstName"> <br/><br/>
  名：<input type="text" v-model="lastName"> <br/><br/>
  全名：<span>{{fullName()}}</span>
</div>
```

3. 使用計算屬性達成

使用方式與 data、methods 相同，一個新的物件屬性`computed`，並且再帶入一個屬性值的物件。

```js
...

methods: {
 demo(){ 
 }
},
computed:{
 fullName:{
  //get有什麼作用？當有人讀取fullName時，get就會被調用，且返回值就作為fullName的值
  //get什麼時候調用？1.初次讀取fullName時。2.所依賴的數據發生變化時。
  get(){
  console.log('get被調用了')
  // console.log(this) //此處的this是vm
  return this.firstName + '-' + this.lastName
  },
  //set什麼時候調用? 當fullName被修改時。
  set(value){
  console.log('set',value)
  const arr = value.split('-')
  this.firstName = arr[0]
  this.lastName = arr[1]
  }
 }
}

...
```

##### 總結

計算屬性：

1. 定義：要用的屬性不存在，要通過已有屬性計算得來。
2. 原理：底層藉助了 Objcet.defineproperty 方法提供的 getter 和 setter 。
3. get 函數什麼時候執行？
  (1).初次讀取時會執行一次。
  (2).當依賴的數據發生改變時會被再次調用。
4. 優勢：與 methods 實現相比，內部有緩存機制（複用），效率更高，找 BUG 方便。
5. 備註：
        (1)計算屬性最終會出現在 vm 上，直接讀取使用即可。
        (2)如果計算屬性要被修改，那必須寫 set 函數去響應修改，且 set 中要引起計算時依賴的數據發生改變。

簡寫樣式：
如果計算屬性的值只要讀取不用修改(不用設定 setter )，則可以使用簡寫方式

```js
computed:{
//完整写法
/* fullName:{
get(){
console.log('get被调用了')
return this.firstName + '-' + this.lastName
   },
   set(value){
    console.log('set',value)
    const arr = value.split('-')
    this.firstName = arr[0]
    this.lastName = arr[1]
   }
  } */
  //簡寫方式
  fullName(){
   console.log('get被调用了')
   return this.firstName + '-' + this.lastName
  }
 }
```
