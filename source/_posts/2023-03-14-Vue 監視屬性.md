Vue  監視屬性，顧名思義就是當值改變時就會執行的 function。

<!--more-->

------

## 基本監視屬性

基本用法：

寫在 Vue 實例內後，並定義要監視的變數，這裡用`isHot`示範
此`isHot`為配置物件，裡面需要設定對應的回傳設定

這邊定義其函數`handler`，當`isHot`發生改變時則會執行函數

`handler`中傳入兩個參數，第一個為改變後的值，第二個為舊的值

設定項 **immediate** 為 true 時會在網頁載入設置監聽器時就調用一次`handler`函數

**immediate** 如果沒有配置時則預設值為 false

```javascript
const vm = new Vue({
   el:'#root',
   data:{
    isHot:true,
   },
   watch:{
    isHot:{
     immediate:true, //初始化時讓handler調用一下
     //handler什麼時候調用？當isHot發生改變時。
     handler(newValue,oldValue){
      console.log('isHot被修改了',newValue,oldValue)
     }
    }
   }
  })
```

另一種寫法，寫在實例外

```javascript
const vm = new Vue({
   el:'#root',
   data:{
    isHot:true,
   }
   ...省略
  })

vm.$watch('isHot',{
    immediate:true, 
    handler(newValue,oldValue){
    console.log('isHot被修改了',newValue,oldValue)
})

```

### 結論

監視屬性watch：

1. 當被監視的屬性變化時, Callback 函數自動調用, 進行相關操作
2. **監視的屬性必須存在，才能進行監視！！**
3. 監視的兩種寫法：
    new Vue 時傳入 watch 配置
    通過 vm.$watch 監視

## 深度監視屬性

假設一個情境，今天 data 中的一個變數 numbers 為一個物件，我想要監視其中改變的屬性要怎麼做呢

```javascript
data:{
    isHot:true,
    numbers:{
     a:1,
     b:1,
    }
   },
```

如果我要監視特定 numbers 內的 a 值可以這樣子寫

**需要加上引號**，因為在 JS 中的 key 寫成`number.a`是不合法的寫法

```javascript
watch:{
    isHot:{         //這段可以先不用看
     handler(newValue,oldValue){
      console.log('isHot被修改了',newValue,oldValue)
     }
    },
    "numbers.a":{
     handler(){
      console.log('a被改變了')
     }
    }
   }
```

可是如果 numbers 內有很多資料，需求改成我要監視裡面的屬性就不能這樣子一個個寫

如果設定為監測 numbers 這個物件則不會發生任何變化，因為監測值是 numbers 這個變數本身存在記憶體的位置

假設：numbers={a:1,b:1} 當 number={a:2,b:2} 時，此時 watch 監測的 numbers 才會發生改變

```javascript
watch:{
    //這段可以先不用看
    isHot:{         
     handler(newValue,oldValue){
      console.log('isHot被修改了',newValue,oldValue)
     }
    },
    //這樣寫除非重新賦值整個 numbers 否則永遠監視不到
    numbers:{
     handler(){
      console.log('a被改變了')
     }
    }
   }
```

此時可以新增一個配置屬性`deep`

```javascript
    //此時不管物件內的哪個層級變數發生變化都可以監視到
    numbers:{
    deep:true
     handler(){
      console.log('a被改變了')
     }
    }
   }
```

### 結論

深度監視
    1. Vue 中的 watch 預設不監測對象內部值的改變（第一層）。
    2. 配置 deep:true 可以監測對象內部值改變（多層結構）。
備註：
    1. Vue 自身可以監測對象內部值的改變，但 Vue 提供的 watch 預設不可以！(效能考量)
    2. 使用 watch 時根據資料的具體結構，決定是否採用深度監視。

## 監視屬性簡寫

如果是正常寫法，想要包含設定選項則是這樣寫

```javascript
watch:{

    //正常寫法
    isHot:{
     // immediate:true, //初始化時讓handler調用一下
     // deep:true,//深度監視
     handler(newValue,oldValue){
      console.log('isHot被修改了',newValue,oldValue)
     }
    },

    //簡寫
    isHot(newValue,oldValue){
     console.log('isHot被修改了',newValue,oldValue,this)
    }
   }
```

如果使用外部`$watch`方式則是這樣寫

```javascript
//正常寫法
  vm.$watch('isHot',{
   immediate:true, //初始化時讓handler調用一下
   deep:true,//深度監視
   handler(newValue,oldValue){
    console.log('isHot被修改了',newValue,oldValue)
   }
  })

  //簡寫
  vm.$watch('isHot',(newValue,oldValue)=>{
   console.log('isHot被修改了',newValue,oldValue,this)
  })
```

使用簡寫的的前提是沒有要配置 immediate 、deep 等配置選項
