紀錄 Chrome 擴充插件開發到上架的心得。
<!--more-->

------

在 Chrome 上開發非常簡單，只要創建幾個檔案就可以開始測試了。
簡單程度甚至像當初剛開始學習網頁一樣。

比較大的差異點是呼叫的Chrome API需要特定權限才可以，這也是上架到商店的審核機制之一。

採到比較大的坑應該是版本問題

目前要上架到商店，環境變數設定檔需要更新到V3版本其格式如下

```json
{
    "manifest_version": 3,
    "name": "LittleIP",
    "version": "1.0.0",
    "description": "My little IP",
    "action": {
      "default_icon": { 
      "16": "icons/ip16.png",
      "32": "icons/ip32.png",
      "48": "icons/ip48.png",
      "128": "icons/ip128.png"
      },
      "default_popup": "popup/popup.html"  
    },
    "content_scripts": [ 
        {
          "matches": ["<all_urls>"],
          "js": ["content_script/base_script.js"]
        }
      ],
      "permissions": [
        "activeTab"
      ]
}

```
原本的 icon 和 popup 定義包在 action 底下，稍微修改一下就可以送交審核了。

練習用專案

[LittleIP](https://github.com/aliceric27/LittleIP)

這個專案簡單的用了兩支公用 API 實現
`ipify.org`這支 API 可以取得當前使用者呼叫的 IP 來源

`ip-api.com` 這支 API 可以用來取得 IP Who-is 相關資訊

並簡單反查當前網站位置，修改 emoji 字體來顯示目前網域所在區域與國家資訊
