拆解以下程式碼其實可以分為兩步驟

在 function 執行期間可以分為兩個階段，首先會先初始化相關參數
使用文章內的 VO 概念建立一個初始化物件

然後才開始執行 function ，此時的 v 為10當然會印出的就是10。

將題目轉換一下，套用剛才的方式很簡單的就可以知道印出的分別為10和3。

首先會先將需要的資料宣告完成這樣理解作用域則為

因為在 function 作用域內使用 let 宣告時會發現已經有一個全域變數 v 已經被宣告，此時在宣告時會報錯

我的答案

一樣是undifiened，此時global區域有一個宣告 a 並且值為4
test() 作用域內有重複宣告的動作，因此 可以理解為 首先先宣告了 a 且值1
不過開始執行fn後又重複宣告一次 a ，因為已經有 a 了，所以值還是保持為1，不過接下來就執行賦值並且值為0
而 b 會因為宣告後直接呼叫，所以還是undifened，然後又將b賦值了2

你的解釋非常準確！ 在這段程式碼中，JavaScript的作用域規則導致了這些變數的值和存取行為。 全域作用域中的變數 a 和函數 test 內部的變數 a 具有不同的作用域，因此它們不會互相干擾。

在呼叫test(1) 後，內部函數test 中的a 變數確實經歷了多次聲明，但由於它們位於不同的作用域中，它們不會相互覆蓋，因此最終結果是a 的值為1（在函數 內部），而b 的值為undefined（因為它在賦值之前被console.log 呼叫）。 然後 b 被賦值為2。 非常清楚的解釋，感謝你的分享！ 如果你還有其他問題，隨時提出。

我的答案

首先會先宣告 x 和 y 在 foo()內，不過嘗試在賦值前呼叫會導致reference error因為第一行的時候此時x值為undifiened

ChatGPT

以下僅供參考

你的答案是正確的！ 在這段程式碼中，變數 x 和 y 雖然在函數 foo() 內部被聲明，但在它們的聲明之前嘗試使用它們會導致引用錯誤（ReferenceError）。 這是因為 let 宣告具有區塊級作用域，因此在宣告之前存取這些變數會觸發暫時性死區（Temporal Dead Zone，TDZ），在 TDZ 內嘗試存取變數會導致參考錯誤。

因此，在第一行嘗試存取 x 時，它仍然在 TDZ 中，因此會引發引用錯誤。 只有在後面的程式碼中才為 x 賦予了值，才能安全地存取它。

CSS 基本選擇器

上面是比較基本的選擇器，也覆蓋 8 成以上使用情境

CSS + 和 ~ 同層選擇器

這兩個選擇器只有在同層下才會有作用

如果將 class 改為+則會選擇在設定條件的同一層的下一個符合的元素來做對應改變。
如果同層中間有其他元素中斷了這個規則，那他就不會被選擇到

CSS權重

基本上 CSS 權重區分如下

!>id>class>tag

CSS 會將所有權重下去做計算

套用的規則會根據 CSS 設定的權重下去評分，如過評分都一樣則 CSS 樣式會由後面的設定覆蓋前面的設定

以下是簡單分數評斷

沒有任何定義的權重分是這個樣子
ID 0 , Class 0 , tag 0
假設最前方 ID 為 1 則後方都不用比較，權重一定最高
ID 1 , Class 0 , tag 0

可以想像一個 ID 是百元鈔、一個 class 是十元硬幣、tag為一元硬幣這樣下去比較

舉一個簡單的例子，假設以下都是定義同一個物件最後顏色會是 blue
因為 ID 權重為 1，下面的三個 class 加起來為3還是被 ID 權重蓋過去

用剛剛的貨幣理論， 100 元 > 30 元這種感覺

相反的如果權重都相同，就會用後面設定的元素覆蓋前面設定的元素

最後有個例外千元鈔，那就是inline-style
這個權重是蓋在ID上方的，新增完這個權重則計算方式變為下表
inline-style > ID > class > tag
