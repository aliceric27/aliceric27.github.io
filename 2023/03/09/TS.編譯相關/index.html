Typescript 編譯相關

發表於 2023-03-09 更新於 2024-03-06 分類於 Javascript ， Typescript

學習 Typescript 下 tsconfig.json 的編譯相關方式

tsconfig.json 設定檔

tsconfig.json 是 ts 編譯器的設定檔，ts編譯器可以根據訊息對程式碼進行編譯

檔案相關

  • includ 用來指定那些 ts 文件需要被編譯

預設值：[“**/*”]

* 表示任意檔案
** 表示任意資料夾

  • exclude 表示不包含哪些檔案

預設值：[“node_modules”, “bower_components”, “jspm_packages”]

compilerOptions 相關

compilerOptions 底下有非常多的設定屬性，每個屬性分別代表不同的意思

  • target

用來指定 ts 編譯的目標版本

接受值：ES3（預設）、ES5、ES6/ES2015、ES7/ES2016、ES2017、ES2018、ES2019、ES2020、ESNext

  • module

用來指定模組化的規範

接受值：CommonJS、UMD、AMD、System、ES2020、ESNext、None

  • lib

用來指定程式碼運行包含的函式庫，通常不會動到

接受值：ES5、ES6/ES2015、ES7/ES2016、ES2017、ES2018、ES2019、ES2020、ESNext、DOM、WebWorker、ScriptHost …

  • outDir

編譯後檔案所在的目錄，預設情況下，編譯後的 js 檔案會和 ts 檔案位於相同的目錄，設定 outDir 後可以改變編譯後檔案的位置

設定後編譯後的 js 檔案將會產生到 dist 目錄

  • allowJs

是否對js文件編譯
預設值：false
接受值：true | false

  • checkJs

檢查 js 程式碼是否符合 ts 規範
預設值：false
接受值：true | false

  • removeComments

編譯完成後是否移除註解
預設值：false
接受值：true | false

  • noEmit

不產生編譯後的文件
預設值：false
接受值：true | false

  • noEmitOnError

當出現錯誤時不產生編譯文件
預設值：false
接受值：true | false

程式碼檢查語法相關

嚴格檢查

  • strict

所有嚴格檢查的總開關，會將所有嚴格檢查設定為 true
預設值：false
接受值：true | false

  • alwaysStrict

用來設定編譯後的文件是否使用嚴格模式
預設值：false
接受值：true | false

  • noImplicitAny

不允許程式碼出現隱式any的寫法
禁止隱式any類型
預設值：false
接受值：true | false

  • noImplicitThis

不允許程式碼出現隱式 this 的寫法(不明確的this類型，即未宣告類型)
預設值：false
接受值：true | false

  • strictNullChecks

嚴格檢查空值
預設值：false
接受值：true | false

範例程式碼，如果在編譯器中 box1 值為空值時則可能不會錯誤，開啟strictNullChecks就會檢查除錯誤，因為 box1 不存在
