Vue2 Basic

發表於 2023-03-03 分類於 Vue

學習 Vue 2 基礎的筆記，以每個小節做一個概念統整

初識 Vue

  • 想讓 Vue 工作就必須創建（new）一個 Vue 物件
  • HTML 容器內仍然遵循 HTML 規範，只是混入一些 Vue 的語法
  • 同一個 Vue 的 HTML 容器與一個 Vue 的物件互相綁定，為一對一對應
  • HTML 中的 {{}}內可以寫入所有 Js 表達式
  • 一旦 Vue 物件的 data 發生改變，HTML 容器內使用的數值會隨時自動更新

Vue 模板語法

插值語法

功能：用於解析 Vue 物件 data 的設定值

寫法： ，xxxx 為 Js 表達式，可以直接讀取到 data 的所有屬性

指令語法

功能：用於解析標籤（包括：標籤屬性、標籤體內容、綁定事件…..）。

舉例：v-bind:href="xxx" 或簡寫為 :href="xxx"，xxx同樣要寫 Js 表達式，

且可以直接讀取到 data 中的所有屬性。

Vue中有很多的指令，且形式都是：v-????，此處只是拿v-bind舉個例子。

資料綁定

Vue中有2種資料綁定的方式：

單向綁定 (v-bind)：資料只能從data流向頁面。

雙向綁定 (v-model)：資料不僅能從data流向頁面，還可以從頁面流向 data。

  • 雙向綁定一般都應用在表單類元素上（如：input、select等）

  • v-model:value 可以簡寫為 v-model，因為 v-model 預設收集的就是value值。

data 與 el 的2種寫法

el 有2種寫法

  1. new Vue 時候配置 el 屬性。
  1. 先 new Vue，隨後再通過vm.$mount('#root')指定 el 的值。

data 有2種寫法

  1. 對象式
  1. 函數式

​ 如何選擇：目前哪種寫法都可以，以後學習到組件時，data 必須使用函數式，否則會報錯。

由Vue管理的函數，一定不要寫箭頭函數，一旦寫了箭頭函數，this就不再是Vue實例了。

MVVM模型

MVVM模型

  1. M：模型(Model) ：data 中的資料
  2. V：視圖 (View) ：模板代碼
  3. VM：視圖模型(ViewModel)：Vue 實例

觀察發現：

  1. data 中所有的屬性，最後都出現在了 vm 身上。
  2. vm 身上所有的屬性及 Vue原型上所有屬性，在 Vue 模板中都可以直接使用。

資料代理

學習 Object.defineProperty()這個內建函數，

這個函數需要傳入3個參數，物件本身、需要定義的值(字串)、設定參數

其中最重要的屬性為 get()、set()

get()替物件參數設定一個 function，當訪問這個參數時隨即 return 對應 function 回傳值

set()當物件參數改變數值時，調用 set()所設定的 function

Vue 當中也用到了這個特性，當我們 new Vue 後，並賦值給一個變數，則會將 Vue 內的 data 物件傳到這個變數中，所以兩邊數值是一模一樣的，只是傳入後 Vue 會幫我們加上 getter 與 setter 等方法，所以我們才可以使用模板函數直接讀取變數 。

存到變數中的 data 會以 _data 命名，所以模板輸入 ，也是可以讀取到該值

7. 事件處理

  • 使用v-on:xxx 或 @xxx 綁定事件，其中xxx是事件名

  • 事件的回調需要設定在 methods 物件中，最終會在 vm 上

  • methods 中設定的函數，不要用箭頭函數！否則 this 就不是 vm 了

  • methods中設定的函數，都是被 Vue 所管理的函數，this 的指向是 vm 或組件實例物件

  • @click=”demo” 和 @click=”demo($event)” 效果一致，但後者可以傳入參數

事件基本用法

使用v-on綁定事件，並填入調用方法名稱(字串)

事件綁定縮寫寫法為@

預設調用方法會傳入event事件(如果沒有傳入任何參數)

如果想要傳入參數則加上括號並傳入指定參數，想要保留預設 event 事件則需要傳入 $event

方法於 Vue 物件中的寫法：

需寫在 Vue 實例內的 methods 中，當 HTML 觸發方法時，Vue 會直接調用對應方法

事件修飾符號

修飾符依照功能可以大概分為下面兩類：

事件修飾符: 事件相關的處理或設定。

按鍵修飾符: 鍵盤或是按鈕的事件中觸發的按鍵設定。

Vue中的事件修飾符：

  1. prevent：阻止預設事件（常用）

  2. stop：阻止事件冒泡（常用）

  3. once：事件只觸發一次（常用）

  4. capture：使用事件的捕獲模式

  5. self：只有 event.target 是當前操作的元素時才觸發事件

  6. passive：事件的預設行為立即執行，無需等待事件回調執行完畢

Vue中常用的按鍵別名：

  • 回車 => enter
  • 刪除 => delete (捕獲“刪除”和“退格”鍵)
  • 退出 => esc
  • 空格 => space
  • 換行 => tab (特殊，必須配合keydown去使用)
  • 上 => up
  • 下 => down
  • 左 => left
  • 右 => right

基本用法

Vue未提供別名的按鍵，可以使用按鍵原始的 key 值去綁定，但注意要轉為 kebab-case（烤肉串命名）

系統修飾鍵（用法特殊）：ctrl、alt、shift、meta

(1).配合 keyup 使用：按下修飾鍵的同時，再按下其他鍵，隨後釋放其他鍵，事件才被觸發。
(2).配合 keydown 使用：正常觸發事件。

也可以使用 keyCode 去指定具體的按鍵（不推薦）

keyCode 已漸漸被瀏覽器棄用，且考量鍵盤規格不同不建議使用

Vue.config.keyCodes.自定義鍵名 = 鍵碼，可以去定製按鍵別名
